Define domains for MSSPs
Supported in:
Google secops
SOAR
The
Domains
page lets you do the following actions:
Create and modify specific domains as internal domains
:
  when an entity is created during the alert ingestion process, it appears as 
  an internal asset. This is especially
  relevant for Managed Security Service Providers (MSSPs) and their customers.
Import or export multiple networks
: You can use the downloaded
  template format to add domains, and then re-import them. This can be useful if
  you're a new customer and want to import existing information into the platform.
  You can use
Export
to create a comprehensive backup of your entire SOAR configuration.
Create a domain
To create a domain, follow these steps:
Go to
Settings
>
Environments
>
Domains
.
On the
Add Domain
page, enter a domain name and select an environment.
Click
Add
.
Edit a domain
To edit a domain, follow these steps:
Go to
Settings
>
Environments
>
Domains
.
Select a domain line and click
edit
Edit a Domain
.
In the
Edit Domain
dialog, make your changes and click
Save
.
Download a domain template
You can download a template to create or modify integrations. It provides
a CSV file containing all the necessary definitions, scripts, and
configuration files.
To download a template, follow these steps:
Go to
Settings
>
Environments
>
Domains
.
Select a domain line and click
download
Download Template
.
Import or export a domain
To import or export a domain, follow these steps:
Go to
Settings
>
Environments
>
Domains
.
Select a domain line and click one of the following:
Click
login
Import
.
Click
Export
.
