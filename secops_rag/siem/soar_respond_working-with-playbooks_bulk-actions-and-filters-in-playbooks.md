Manage bulk actions and filters in playbooks
Supported in:
Google secops
SOAR
This document explains how to use bulk actions to perform simultaneous tasks on
multiple playbooks and playbook blocks, boosting your productivity.
The following actions are available on the
Playbooks
page:
add
Add New Playbook or Block
: Selects the folder and environment to
which the playbook or playbook block belongs.
Edit
: Selects single or multiple playbooks and blocks for use
    with the action menu.
filter_alt
Open Filter
:  Opens the
Playbooks Filters
dialog where you can choose to filter the display based
on:
Playbook simulator is on or off
Active/inactive playbooks, also referred to as
Enabled/Disabled
Priority level
Environments (multi-select option)
format_list_bulleted
Menu
: Displays the following actions. Before you click this menu, click
edit
Edit
and select the required playbooks or playbook blocks.
create_new_folder
New Folder
: Adds a new folder. To edit the folder name, click
edit
Edit
, click
New Folder
, enter the
    new name, and then click
keyboard_arrow_up
Up arrow
.
content_copy
Duplicate
: Creates a duplicate playbook with the following options:
Keep or Change Priority
: Keeps the original
        priority value, or changes to a different priority
Keep in same folder or move to a different folder
Choose environments it belongs to
: options include
single
,
multiple
,
         or
all
environments, where
all
refers to both current and
         future-defined environments.
Change priority
: Changes the priority value.
Export/Import
: Sends both playbooks and playbook
    blocks from staging to a production server and the other way around. Playbooks
    are exported with views. The system only recognizes ZIP files for
    import.
Move To
: Moves playbooks and playbook blocks to another folder or
    create a new folder from this option.
Delete
: Deletes playbooks or playbook blocks.
Show All
: Shows playbooks or playbook blocks, or both in the queue.
